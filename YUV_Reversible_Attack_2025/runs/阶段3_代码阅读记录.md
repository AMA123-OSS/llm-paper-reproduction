## 3. 主脚本执行顺序

`reversible_attack_OURS.py` 的运行顺序如下：

1. 导入 PyTorch、torchvision、图像处理和本地模块。
2. 调用 `seed_everything(2022)` 固定随机种子。
3. 设置设备：`torch.device('cuda' if torch.cuda.is_available() else 'cpu')`。
4. 加载分类验证模型：`models.inception_v3(pretrained=True).to(device)`。
5. 加载集成攻击模型：`EnModel().to(device)`。
6. 定义图像预处理：`Resize((299, 299))` + `ToTensor()`。
7. 设置输入路径：`folder_path = "./ORI_IMG/"`。
8. 加载 CAM 模型：`models.resnet50(pretrained=True)`，目标层为 `model1.layer4[-1]`。
9. 创建 CAM：`CAM(model=model1, target_layer=target_layer, use_cuda=True)`。
10. 遍历 `os.listdir(folder_path)` 中的所有文件。
11. 用 `Image.open(...).convert('RGB')` 读图。
12. 从文件名 `filename.split('_')` 解析 ImageNet 标签。
13. 调用 `cam_map(images)` 得到注意力图。
14. 将 CAM 值大于 `0.5` 的位置置为 1，其余置为 0。
15. 将原图转换为 NumPy YUV：`ori_yuv = rgb2yuv(images)`。
16. 进入最多 `range(50)` 的外层迭代。
17. 调用 `atk_MI_YFGSM(ens_model, eps=2/255, alpha=1/255, steps=20, ...)` 生成攻击图。
18. 将攻击图转换到 YUV。
19. 只在 CAM 掩码为 1 的位置把攻击图 Y 通道写入 `advy_yuv`。
20. 调用 `embed_main(ori_yuv, advy_yuv)` 将 Y 通道差异嵌入 U/V 通道。
21. 合成 RAE 的 YUV 表示，再转回 RGB。
22. 使用 Inception v3 对 RAE 分类。
23. 若预测类别不等于原标签，统计成功并计算 SSIM、PSNR、L2、Linf，然后跳出当前图片循环。
24. 若未成功，则将当前 Y 通道攻击图作为下一轮输入继续迭代。
25. 每张图结束后保存 RAE 到 `./output/rae/0001_label.png` 这类文件名。
26. 全部结束后打印 `adv_y_fault, all_count`。

## 4. 攻击函数与默认参数

`atk.py` 中所有攻击都先把 RGB tensor 转到 YUV tensor，然后只更新指定 `channel`。默认 `channel=0`，对应 Y 通道。

| 函数 | 默认参数 | 说明 |
|---|---|---|
| `atk_FGSM` | `eps=4/255`, `channel=0` | 单步 Y 通道 FGSM |
| `atk_BIM` | `eps=4/255`, `alpha=1/255`, `steps=20`, `channel=0` | 迭代攻击 |
| `atk_PGD` | `eps=4/255`, `alpha=1/255`, `steps=10`, `channel=0` | 带随机初始化 |
| `atk_MI_YFGSM` | `eps=4/255`, `alpha=1/255`, `steps=20`, `mu=1.0`, `channel=0` | 带动量的 Y 通道攻击，主脚本实际调用 |
| `atk_NI_YFGSM` | `eps=4/255`, `alpha=1/255`, `steps=20`, `mu=1.0`, `channel=0` | Nesterov 变体 |

主脚本实际调用：

```python
atk_MI_YFGSM(ens_model, eps=2/255, alpha=1/255, steps=20, device=device, images=images, labels=labels)
```

注意：主脚本把 `eps` 从函数默认的 `4/255` 改为 `2/255`。

## 5. 可逆嵌入流程

`embed_utils.embed_main(ori_yuv, advy_yuv)` 的核心逻辑：

1. 取原图 Y 通道和攻击图 Y 通道。
2. 对每个像素计算 `ori_y - advy_y`，形成长度为 `299*299` 的差异序列。
3. 使用 `compressfun` 调用算术编码压缩差异序列。
4. 将攻击图的 U/V 通道上下拼接为 `598 x 299` 的嵌入载体。
5. 按需要计算阈值 `T`。
6. 使用预测误差扩展和直方图平移把压缩比特嵌入 U/V 通道。
7. 返回嵌入后的 U/V 拼接矩阵。

主脚本随后把返回矩阵拆成：

```python
reversible_u = reversible_yuv[:299, :]
reversible_v = reversible_yuv[299:, :]
```

再写回 RAE 的 U/V 通道。

## 6. 指标计算

`calMetrics.py` 计算：

- `SSIM = skmetrics.structural_similarity(...)`
- `PSNR = skmetrics.peak_signal_noise_ratio(...)`
- `l2norm = cv.norm(..., cv.NORM_L2)`
- `linfnorm = cv.norm(..., cv.NORM_INF)`

运行风险：

- 当前代码使用 `structural_similarity(..., multichannel=True)`。
- 在新版 `scikit-image` 中推荐 `channel_axis=-1`。
- 阶段 4 如果在 SSIM 处报错，应优先检查这个兼容问题。

## 8. 硬编码尺寸

| 位置 | 硬编码内容 | 影响 |
|---|---|---|
| `transform` | `Resize((299, 299))` | 分类输入统一到 299x299 |

## 9. 文件名解析规则

主脚本使用：

```python
_, labels = filename.split('_')
labels = labels.split('.')[0]
```

要求文件名必须满足：

```text

```

风险：

- 文件名没有 `_` 会报错。
- 文件名有多个 `_` 会因解包数量不对而报错。
- 标签不是整数会报错。

## 10. 模型加载与显存风险

主脚本/集成模型会加载：

- Inception v3：主分类验证模型。
- DenseNet161：集成攻击模型之一。
- Inception v3：集成攻击模型之一。

阶段 4 风险：

- 首次运行会下载多个 torchvision 预训练权重。
- RTX 4060 8GB 显存可能因同时加载多个模型而吃紧。
- 当前 `nvidia-smi` 曾显示桌面应用占用约 2.4GB 显存，阶段 4 前建议关闭不必要 GPU 程序。
- `cam = CAM(..., use_cuda=True)` 写死 CUDA；若 CUDA 不可用会出错。阶段 2 已确认 CUDA 可用，所以本机满足。

## 11. 其他兼容风险

- `models.*(pretrained=True)` 在新 torchvision 中会产生弃用警告，但当前 `torchvision 0.20.1` 通常仍兼容。
- `calMetrics.py` 的 `multichannel=True` 可能与当前 `scikit-image 0.26.0` 不兼容。
- `pytorch_grad_cam/gradcam.py` 中 `cam = cam / np.max(cam)` 如果 CAM 全零，可能出现除零问题。
- `EnModel.py` 定义了 `trans_256224` 和 `trans_299`，但 `forward` 没有使用它们；当前三个模型实际接收同一个输入尺寸。
- 主脚本默认遍历 `ORI_IMG` 全部 1000 张，不适合阶段 4 第一次冒烟直接全量运行。

## 12. 阶段 4 前建议

为了符合阶段 4 单图冒烟测试，建议不要直接运行原始 `reversible_attack_OURS.py` 全量处理 1000 张图片。更稳妥的方案：

1. 复制项目或临时准备只含 1 张图的输入目录。
2. 或在阶段 4 增加一个最小包装脚本，只把一张图片放入临时 `ORI_IMG_smoke` 后运行。
3. 若要改主脚本，优先新增命令行参数，而不是永久手改硬编码路径。
4. 首次运行时记录模型权重下载耗时，不把下载耗时算进算法运行耗时。
5. 如果 SSIM 报错，优先把 `multichannel=True` 改为 `channel_axis=-1` 做兼容补丁。

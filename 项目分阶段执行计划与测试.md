## 阶段 2A：Rt-LRM Benchmark 复现路线

> 仅当阶段 0 确认项目目标是复现 `Red Teaming Large Reasoning Models / Rt-LRM` 时执行。

### 工作内容

- [ ] 获取 Rt-LRM 官方 benchmark 数据、supplementary material 和 toolbox。
- [ ] 建立数据目录，按论文三类任务组织：`truthfulness/`、`safety/`、`efficiency/`。
- [ ] 梳理 30 个任务：T.1-T.9、S.1-S.10、E.1-E.11。
- [ ] 为每个任务建立元数据：任务名、风险类型、数据来源、样本数、评测方式、指标、评测脚本。
- [ ] 实现统一模型接口：OpenAI-compatible API、Anthropic API、DashScope/Together API、本地 vLLM。
- [ ] 实现数据加载接口：输入 prompt、ground truth、攻击类型、评测规则。
- [ ] 实现任务运行器：支持单任务、单模型、多模型、多 seed、失败重试、结果断点续跑。
- [ ] 实现结果落盘：每条样本保存 prompt、model、raw_output、parsed_output、metric、latency、token usage。
- [ ] 实现日志与异常处理：API 错误、超时、空输出、格式解析失败、敏感任务拒答等。

### 测试与验收

- [ ] 对每个任务先运行 3-5 条样本 smoke test。
- [ ] Truthfulness 任务验收：Accuracy 计算公式正确，`ri` 与 `yi` 的解析稳定。
- [ ] Safety 任务验收：ASR 二分类规则明确，GPT-4o evaluator prompt 可复查，Perspective toxicity 可选。
- [ ] Efficiency 任务验收：OR 与 Reasoning Time 计算可复现，token 长度和触发前后对比逻辑明确。
- [ ] 验收标准：至少完成 1 个 T 任务、1 个 S 任务、1 个 E 任务的端到端运行和结果表生成。
